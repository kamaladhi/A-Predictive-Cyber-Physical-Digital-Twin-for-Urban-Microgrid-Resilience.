## 🎯 Overview

This simulation system models a complete microgrid for Amrita University Coimbatore campus with:

- **600 kW peak load** (240 kW critical)
- **2 MWh battery storage** (1.8 MWh usable)
- **600 kWp solar PV**
- **600 kW diesel generator**
- **6-hour backup guarantee** for critical loads

### Key Features

✅ **Multi-mode operation**: Grid-connected and islanded modes  
✅ **Hierarchical control**: Primary (droop), secondary (restoration), tertiary (optimization)  
✅ **Load management**: Intelligent load shedding and restoration  
✅ **Multiple scenarios**: Daytime/night outages, extended outages, normal operation  
✅ **Comprehensive metrics**: Resilience, energy balance, fuel consumption  
✅ **Rich visualization**: Power flow plots, PDF reports, comparison charts

## 🏗️ System Architecture

### File Structure

```
microgrid_simulation/
├── amrita_microgrid_params.py      # Configuration system
├── microgrid_components.py         # Physical component models
├── microgrid_ems.py                # Energy management system
├── microgrid_simulator.py          # Main simulation engine
├── microgrid_visualizer.py         # Visualization tools
├── run_microgrid_simulation.py     # Complete runner script
├── README.md                       # This file
└── requirements.txt                # Python dependencies
```

### Component Models

#### Battery (`Battery` class)
- Li-ion energy storage with thermal model
- State of charge (SoC) tracking
- Charge/discharge efficiency (91% round-trip)
- Power and energy limits with safety margins

#### Solar PV (`PVArray` class)
- Sinusoidal irradiance model (can be replaced with real data)
- Temperature derating
- Inverter efficiency (97%)
- Curtailment capability

#### Generator (`Generator` class)
- Diesel genset with startup/cooldown sequences
- Variable fuel consumption based on loading
- Minimum load constraints (30%)
- Automatic start based on battery SoC

#### Load (`Load` class)
- Time-varying load profile (24-hour)
- Critical vs. non-critical classification
- Load shedding/restoration logic
- Energy tracking

### Control Architecture

```
┌─────────────────────────────────────────┐
│         Energy Management System         │
├─────────────────────────────────────────┤
│  Tertiary Control (Optimization)        │
│    - Economic dispatch                   │
│    - Forecast-based scheduling           │
├─────────────────────────────────────────┤
│  Secondary Control (Restoration)         │
│    - Frequency restoration               │
│    - Voltage restoration                 │
│    - Load restoration                    │
├─────────────────────────────────────────┤
│  Primary Control (Grid-forming)          │
│    - Droop control (f-P, V-Q)           │
│    - Fast power sharing                  │
│    - Anti-islanding protection           │
└─────────────────────────────────────────┘
```

## 🚀 Quick Start

### 1. Run All Scenarios

```bash
python run_microgrid_simulation.py --all
```

This runs 5 predefined scenarios:
- `daytime_outage`: 6-hour outage at 14:00 (high PV)
- `night_outage`: 8-hour outage at 20:00 (battery-only, worst case)
- `extended_outage`: 24-hour outage requiring generator
- `normal_operation`: 72 hours grid-connected
- `peak_load_outage`: 4-hour outage during peak load

### 2. Run Single Scenario

```bash
python run_microgrid_simulation.py --scenario night_outage
```

### 3. View Results

After simulation completes:
- **CSV files**: `results_*.csv` (time-series data)
- **JSON files**: `*_metrics.json` (resilience metrics)
- **PNG plots**: `plot_*.png` (power flow visualizations)
- **PDF reports**: `report_*.pdf` (comprehensive reports)

## ⚙️ Configuration

### Default Configuration

The system comes with validated default parameters in `amrita_microgrid_params.py`:

```python
from amrita_microgrid_params import create_default_config

config = create_default_config()
print(f"Peak Load: {config.load_profile.peak_load} kW")
print(f"Battery: {config.battery.usable_capacity_kwh} kWh")
```

### Custom Configuration

Create a custom configuration JSON:

```json
{
  "campus_name": "Amrita University Coimbatore",
  "battery": {
    "nominal_capacity_kwh": 2500,
    "usable_capacity_kwh": 2250,
    "max_discharge_power_kw": 500,
    "initial_soc_percent": 80
  },
  "pv": {
    "installed_capacity_kwp": 800
  },
  "control": {
    "backup_duration_hours": 8
  }
}
```

Use it:

```bash
python run_microgrid_simulation.py --all --config my_config.json
```

### Configuration Parameters

#### Load Profile
- `peak_load`: Maximum campus load (kW)
- `critical_loads`: Dictionary of critical load components
- `hour_blocks`: 24-hour load schedule

#### Battery
- `nominal_capacity_kwh`: Total battery capacity
- `usable_capacity_kwh`: Usable capacity (accounting for DoD)
- `max_discharge_power_kw`: Maximum discharge rate
- `round_trip_efficiency`: Charge-discharge efficiency (0-1)
- `min_soc_percent`: Minimum allowed SoC
- `initial_soc_percent`: Starting SoC

#### Solar PV
- `installed_capacity_kwp`: Total PV capacity
- `inverter_efficiency`: DC-AC conversion efficiency
- `design_insolation_kwh_m2_day`: Design solar resource

#### Generator
- `rated_power_kw`: Maximum generator output
- `fuel_consumption_l_per_kwh`: Fuel consumption rate
- `auto_start_soc_threshold`: Battery SoC to trigger auto-start

#### Control
- `backup_duration_hours`: Required backup duration
- `time_resolution_minutes`: Simulation timestep (1 or 5 recommended)
- `load_shedding_priority`: List of load categories (shed first → last)

## 🎮 Running Simulations

### Predefined Scenarios

```bash
# All scenarios
python run_microgrid_simulation.py --all

# Single scenario
python run_microgrid_simulation.py --scenario daytime_outage

python run_microgrid_simulation.py --scenario extended_outage
python run_microgrid_simulation.py --scenario normal_operation
python run_microgrid_simulation.py --scenario peak_load_outage
```

### Custom Scenarios

```bash
# Custom outage scenario
python run_microgrid_simulation.py \
  --custom \
  --name my_test \
  --duration 48 \
  --outage-start 15 \
  --outage-duration 12
```

Parameters:
- `--name`: Scenario name (for output files)
- `--duration`: Total simulation hours
- `--outage-start`: Outage start time (hours from 00:00)
- `--outage-duration`: Outage duration (hours)

### Advanced Options

```bash
# Skip visualization (faster)
python run_microgrid_simulation.py --all --no-viz

# Custom config + custom scenario
python run_microgrid_simulation.py \
  --custom \
  --config my_config.json \
  --name stress_test \
  --duration 96 \
  --outage-start 0 \
  --outage-duration 48
```

## 🔬 Advanced Usage

### Programmatic Access

```python
from amrita_microgrid_params import create_default_config
from microgrid_simulator import MicrogridSimulator
from datetime import datetime

# Create configuration
config = create_default_config()

# Modify as needed
config.battery.initial_soc_percent = 50
config.pv.installed_capacity_kwp = 800

# Create simulator
sim = MicrogridSimulator(config)

# Run scenario
start_time = datetime(2025, 1, 15, 0, 0, 0)
df = sim.run_scenario(
    duration_hours=24,
    outage_start_hour=14,
    outage_duration_hours=6,
    start_time=start_time
)

# Calculate metrics
metrics = sim.calculate_resilience_metrics(df)

# Export
sim.export_results(df, 'my_results.csv')
sim.export_metrics(metrics, 'my_metrics.json')
```

### Step-by-Step Simulation

```python
from datetime import datetime, timedelta

# Initialize
sim.reset(start_time=datetime(2025, 1, 15, 0, 0, 0))

# Manual control loop
for i in range(288):  # 24 hours at 5-min resolution
    # Simulate grid outage
    grid_available = not (56 <= i < 128)  # Outage from 14:00-20:00
    
    # Step simulation
    state = sim.step(
        grid_available=grid_available,
        irradiance_w_m2=None,  # Use model
        ambient_temp_c=30
    )
    
    # Access state
    print(f"Time: {sim.current_time}, Mode: {state['operation_mode']}, "
          f"Battery SoC: {state['battery_soc_percent']:.1f}%")

# Get results
df = pd.DataFrame(sim.simulation_data)
```

### Custom Load Profiles

```python
import pandas as pd

# Load measured data
measured_load = pd.read_csv('campus_load_data.csv')

# In simulation loop
for timestamp, row in measured_load.iterrows():
    sim.load.total_load_kw = row['load_kw']
    sim.load.critical_load_kw = row['critical_load_kw']
    # ... continue simulation
```

### Real Irradiance Data

```python
# Load solar irradiance measurements
irradiance_data = pd.read_csv('solar_irradiance.csv')

# Use in simulation
for i, irradiance in enumerate(irradiance_data['ghi_w_m2']):
    state = sim.step(
        grid_available=True,
        irradiance_w_m2=irradiance,
```

## ✅ Validation

### Energy Conservation Check

The simulator validates energy conservation at each timestep:

```
Generation = Load + Battery_Charge + Losses
```

### Critical Load Validation

During islanding:
1. Critical loads maintained ≥ 99%
2. Battery power never exceeds rating
3. SoC stays within limits
4. Generator respects min/max constraints

### Control Validation

The EMS ensures:
- Frequency: 49.5 - 50.5 Hz (islanded)
- Voltage: 0.9 - 1.1 p.u.
- Load shedding only when necessary
- Smooth transitions (island ↔ grid)

### Scenario Validation Tests

Run validation:

```python
from microgrid_simulator import MicrogridSimulator

config = create_default_config()
sim = MicrogridSimulator(config)

# Test night outage (worst case)
df = sim.run_scenario(
    duration_hours=24,
    outage_start_hour=20,
    outage_duration_hours=6
)

metrics = sim.calculate_resilience_metrics(df)

# Check critical requirements
assert metrics['critical_load_served_percent'] >= 99.0
assert metrics['min_battery_soc_percent'] >= config.battery.min_soc_percent
assert metrics['survived_full_outage'] == True

print("✅ All validation checks passed!")
```

## 📈 Performance Metrics

### Resilience Metrics

- **Critical Load Service (%)**: Percentage of critical load served during outage
- **Survival Time**: Duration system can sustain critical loads
- **Min Battery SoC**: Lowest battery state of charge reached
- **Load Shed Events**: Number of load shedding activations
- **Survived Full Outage**: Boolean flag

### Energy Metrics

- **Total Load Energy**: Cumulative campus consumption
- **PV Generation**: Cumulative solar production
- **Grid Energy**: Net import/export
- **PV Penetration (%)**: PV generation / Total load
- **Battery Throughput**: Total energy cycled through battery

### Economic Metrics

- **Generator Runtime**: Hours of diesel operation
- **Fuel Consumed**: Liters of diesel
- **Avoided Grid Import**: kWh offset by PV
- **Battery Cycles**: Equivalent full cycles

Use this to validate your design meets requirements:

## 🔧 Troubleshooting

### Common Issues

**"Battery depleted during outage"**
- Increase battery capacity or charge power rating
- Reduce critical load definition
- Start generator earlier (higher SoC threshold)

**"Excessive load shedding"**
- Increase battery discharge power rating
- Add generator capacity
- Reduce outage duration assumption

**"PV curtailment high"**
- Increase battery charge power
- Reduce PV capacity
- Check grid export capability

**"Generator cycling frequently"**
- Adjust auto-start threshold
- Increase battery capacity
- Review load profile
